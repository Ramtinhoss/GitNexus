# GitNexus v1.3.11 Release Notes

Release date: 2026-03-18
Package: `@veewo/gitnexus`
Channel target: `latest`
Previous stable: `1.3.10`
RC validated: `1.3.11-rc.1`

## Highlights

### Unity resource retrieval hardening

- Moved full binding recovery to query-time hydration to keep indexed data lighter while preserving `context/query --unity-resources on` fidelity.
- Added hydration budgets, lazy overlay backfill, in-flight dedupe, and follow-up optimizations for more predictable Unity retrieval latency.
- Expanded support for serializable-class resource bindings and added parity/equivalence coverage for hydrated results.

### Analyze memory and storage reductions

- Reduced analyze-time memory pressure by streaming serializable type index work, streaming Kuzu relationship pair buckets, bounding CSV cache bytes, and releasing graph state earlier after Kuzu load.
- Compacted persisted Unity binding payloads and added resource summary persistence primitives to keep index output smaller.

### CLI and workflow improvements

- Hardened `analyze` close policy and signal exit handling.
- Improved setup/idempotency coverage for Codex/OpenCode/Claude flows.
- Added release path hygiene checks and refreshed release/distribution runbooks for the scoped package workflow.

### Benchmarks and verification

- Added analyze-memory sampling utilities, Unity lazy-context performance gates, agent-context benchmark coverage, and U2 end-to-end benchmark/report tooling.
- Archived the supporting reports and runbooks used to validate the release candidate.

## 重点更新

### Unity 资源检索增强

- 将完整绑定恢复迁移到查询时 hydration，在保持 `context/query --unity-resources on` 返回能力的同时减轻索引存储负担。
- 增加 hydration 预算、lazy overlay 回填、并发去重和后续优化，使 Unity 检索延迟表现更稳定。
- 扩展 serializable class 资源绑定支持，并为 hydration 结果增加 parity/equivalence 校验覆盖。

### Analyze 内存与存储优化

- 通过串流 serializable type index、串流 Kuzu relationship pair buckets、限制 CSV 缓存字节数，以及在 Kuzu load 后更早释放 graph，降低 analyze 期间的内存压力。
- 压缩持久化的 Unity binding payload，并增加 resource summary 持久化模型，缩小索引体积。

### CLI 与工作流改进

- 加强 `analyze` 的 close policy 与 signal exit 处理。
- 提升 Codex、OpenCode、Claude 配置流程的幂等性和稳定性。
- 增加 release path hygiene 检查，并更新 scoped package 的发布/分发文档。

### Benchmark 与验证能力

- 新增 analyze-memory 采样工具、Unity lazy-context 性能门禁、agent-context benchmark 覆盖，以及 U2 端到端 benchmark/report 工具链。
- 归档了用于验证本次 RC 的报告与 runbook。
